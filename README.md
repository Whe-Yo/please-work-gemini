# Please-Work Harness

어떤 AI 에이전트(Claude Code, Gemini CLI, Cursor, Cline 등)에도 즉시 탑재할 수 있는 **범용 에이전트 하네스** 모음입니다.
규칙(Rules), 스킬(Skills), 외부 도구 명세(MCP)의 순수 재료만을 제공하며, 설치 환경이나 런타임에 종속되지 않습니다.

> [!CAUTION]
> **이 저장소는 도구함(Toolbox)입니다. 도구함 안에서 작업하지 않습니다.**
> 에이전트는 이 저장소에서 재료를 읽고 복사하여 자신의 환경에 주입합니다.
> 프로젝트 작업의 일환으로 이 저장소 내 파일을 수정하거나, 작업 로그를 기록하거나, 설정을 적용하는 행위는 금지입니다.

---

## 이 프로젝트의 의의

### 1. 적응형 하네싱 (Adaptive Harnessing)

에이전트는 고정된 도구 묶음이 아니라, 프로젝트와 환경에 맞게 스스로 스킬을 선택하고 장착합니다.

- **자가 선택**: `setup` 스킬로 에이전트가 `SKILL_INDEX.md`를 읽고 현재 환경에 맞는 스킬만 골라 장착합니다.
- **주기적 갱신**: `manage` 스킬로 스킬 목록을 업데이트하고, 필요 없어진 스킬을 제거합니다.
- **에이전트 비종속**: 모든 스킬은 특정 에이전트 API나 CLI에 의존하지 않습니다. 환경별 대안(서브에이전트 툴, CLI 호출, 새 채팅 안내)을 스킬 내부에서 분기 처리합니다.

### 2. RPW — 현재 상태 스냅샷 (Rule, Plan, Work)

세션이 바뀌어도 맥락이 끊기지 않도록, 단일 문서가 **현재 상태**를 중앙에서 관리합니다. 히스토리 로그가 아니라 스냅샷입니다 — 세 섹션 모두 갱신 시 덮어씁니다. 히스토리는 버전 관리(git)가 담당합니다.

- **규칙(Rule)**: 프로젝트 공통 행동 규칙. 3-tier(Always do / Ask first / Never do)로 명시해 세션마다 재주입합니다.
- **계획(Plan)**: 지금 하는 작업 단위의 목표와 체크리스트. 완료되면 덮어씁니다.
- **작업(Work)**: 지금 어디까지 됐는지 2~3문장 스냅샷. 다음 세션의 `boost`가 즉시 복원합니다.
- **템플릿**: [`rules/rule_plan_work_template.md`](rules/rule_plan_work_template.md)를 새 프로젝트에 복사해 바로 사용합니다.

### 3. n회 안티테제 검토 (n-times Antithesis Review)

같은 세션에서 쌓인 컨텍스트 오버피팅과 확증 편향을 제거하기 위해, 이전 대화를 모르는 독립 인스턴스가 검토를 수행합니다.

- **격리된 컨텍스트**: 검토자는 RPW 문서와 검토 대상만 입력으로 받습니다. 이전 대화 흐름은 전달하지 않습니다.
- **반론 관점**: "이 접근이 맞다"는 전제 없이, 논리 비약·누락·과도한 가정을 찾도록 지시합니다.
- **n회 순차**: 검토 결과를 반영한 결과물을 다음 인스턴스가 재검토합니다. n은 에이전트가 변경 규모에 따라 1~3회로 **자율 판단**합니다(난이도별 자동 조정 같은 고정 규칙은 두지 않습니다 — 최소 1회).
- **구현**: `antithesis` 스킬이 환경별로 자동화(서브에이전트 툴) 또는 수동 안내(새 채팅 + 프롬프트 복붙)로 분기합니다.

## 구성 요소

### 지능 재료: 글로벌 규칙셋 ([rules/AGENTS.md](rules/AGENTS.md))
에이전트의 사고·검증·보고 절차를 강제하는 범용 행동 원칙. 에이전트 룰 파일(AGENTS.md, CLAUDE.md, .cursorrules 등)에 주입하거나, 에이전트 시스템 프롬프트에 붙여넣어 사용합니다.

### 행동 재료: 스킬 11종 ([skills/](skills/))
상황에 따라 에이전트가 로드해 자율 수행하는 절차 지침서. 전체 목록은 [`skills/SKILL_INDEX.md`](skills/SKILL_INDEX.md)를 참고합니다.

### 도구 재료: MCP 템플릿 ([mcp/mcp_template.json](mcp/mcp_template.json))
지식 컷오프 해결과 외부 API 환각 방지를 위한 MCP 서버 명세 (context7, sequential-thinking, exa, memory).

## 개발자 디버그 로그 ([log_for_test/](log_for_test/))

안티테제 검토 결과, 스킬 실증 기록, 설계 결함 발견 내역을 보관합니다. 파일명 형식은 `YYMMDD_HHMM_에이전트명.md`. 프로덕션 동작과 무관하며, 하네스 자체를 개발·개선할 때 참고용으로 사용합니다.

## 하네스의 두 층 — 효력의 경계

이 하네스는 두 층으로 구성됩니다. 범용성의 한계와 강제력의 위치를 정직하게 구분하기 위함입니다.

| 층 | 구성 | 범용성 | 효력 |
| :--- | :--- | :--- | :--- |
| **지시 층** | AGENTS.md, 스킬, RPW (프롬프트·마크다운) | 모든 에이전트 | 소프트 — "대체로 따름" |
| **강제 층** | Hooks, 권한 게이트, 샌드박스 (런타임 종속) | 에이전트별 어댑터 | 하드 — "위반 불가" |

- **지시 층**이 가치의 대부분을 짊어집니다. 에이전트가 규칙을 읽고 대체로 따릅니다. 단 가끔 미끄러집니다.
- **강제 층**은 치명적 소수 규칙(.env 보안, 파괴적 작업, 루프 폭주)만 하드 강제합니다. 강제 수단이 있는 에이전트(예: Claude Code의 Hooks·permissions)에 얇은 어댑터로 바인딩합니다.
- 효력은 On/Off가 아니라 신뢰도 그라데이션입니다. "에이전트를 특정해야 효력이 생긴다"가 아니라, **마지막 가장 위험한 일부를 보장하려면 어댑터가 필요하다**가 정확한 그림입니다.

> 치명적 규칙은 양쪽에 둡니다 — 지시 층(텍스트로 모든 에이전트에 전달) + 강제 층(가능한 에이전트에서 실제 차단).

## 적용 방법

1. [`rules/AGENTS.md`](rules/AGENTS.md) 내용을 에이전트의 룰 파일에 주입합니다.
2. 에이전트에게 `setup` 스킬을 실행하도록 지시합니다. 이후 스킬 선택·장착·MCP 등록·사용자 설정 확인은 에이전트가 스스로 처리합니다.

> RPW 문서는 프로젝트별로 생성됩니다. 에이전트가 새 프로젝트에서 처음 실행될 때 해당 프로젝트 루트에 자동으로 생성합니다.
