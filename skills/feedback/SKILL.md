# Feedback — 사용 중 피드백 기록

하네스를 다른 프로젝트에 쓰다 발견한 문제를 저장소의 `log_for_test/`(성역의 **유일한 쓰기 예외**)에 md로 남긴다. 사용자가 "피드백/의견 남겨"라고 하면 아래를 자동 수행한다.

## 절차

1. **정리** (추측 말고 실제 일어난 것만): 하려던 것 / 일어난 것 / 기대한 것, 관련 스킬·규칙, 환경(에이전트·OS·도구), 재현 절차, 유형(`bug`/`enhancement`/`friction`).

2. **범위·비밀 점검** (둘 다 필수):
   - **하네스에 관한 것만** 남긴다. 프로젝트 작업 내용·분석 결과를 여기 커밋하지 않는다 (무관 프로젝트 내용을 잘못 커밋해 revert한 사고 이력 있음).
   - `.env` 값·토큰·자격증명·`.env` 파일 내용을 절대 포함하지 않는다.

3. **클론 위치 확인**: 이 에이전트의 `~/.agents/harnessing_state_{에이전트}.json`의 `repoPath`(하네스 클론 — 현재 프로젝트 작업 클론과 별개).

4. **파일 작성**: `{repoPath}/log_for_test/{YYMMDD_HHMM}_{에이전트}_feedback.md`. `_feedback` 표식으로 검토 로그와 구분한다. 형식은 아래.

5. **커밋·푸시 (안전하게)**:
   - **그 파일만** `git add <파일경로>` — `git add -A`/`.` 금지(다른 변경 끌어오지 않기).
   - 푸시: 거부되면 그 피드백 커밋만 `git pull --rebase` 후 재시도. **최대 2회**(AGENTS 4절), 그래도 실패하면 중단하고 md 본문을 사용자에게 출력(수동 처리 안내).
   - 하네스 클론에 커밋 안 된 다른(하네스) 작업이 있으면 자동 커밋하지 말고 사용자에게 먼저 안내한다.

6. **환경 분기**: git/클론/push가 불가하면 아래 형식의 md 본문을 출력해 사용자가 직접 배치하게 한다. **삭제제한 마운트(Cowork/Claude Desktop 등)**에서는 `git status`가 남긴 `.git/index.lock`이 unlink 불가로 잔존해 이후 git이 전부 막힐 수 있다 → 삭제권한 확보(Cowork는 파일 삭제 승인) 후 잔존 락 제거, 그래도 불가하면 md 출력으로 폴백.

## 형식
```
# Feedback — {YYMMDD_HHMM} / {에이전트}
## 유형
bug | enhancement | friction
## 요약
(한 줄)
## 무슨 일이
- 하려던 것 / 일어난 것 / 기대한 것
## 환경
- 에이전트 / OS / 도구
## 재현
1.
## 관련 스킬·규칙
```

## 규칙
- 성역: `log_for_test/` 외 경로는 수정 금지. 피드백은 이 폴더에만.
- main 직접 커밋은 **프로토타입 한정 허용**. push 경쟁이 잦아지면 `feedback/{agent}` 브랜치+PR로 전환(ROADMAP 참조).
- 비밀·프로젝트 작업 내용 미포함.
