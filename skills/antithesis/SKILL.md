# Antithesis — 독립 컨텍스트 안티테제 검토

이전 대화 흐름에 오염되지 않은 새 AI 인스턴스가 RPW 문서만을 기반으로 검토한다.
목적: 현재 대화에서 쌓인 컨텍스트 오버피팅과 확증 편향으로 놓친 논리적 오류를 탐지.

## 절차

1. **컨텍스트 수집**: RPW 문서(rule_plan_work.md 또는 work.md) 전체 + 검토 대상(diff, 코드, 설계안).

2. **격리된 인스턴스 생성** (환경별):
   - **서브에이전트 툴이 있는 경우** (예: Claude Code의 Agent 툴): 현재 대화 컨텍스트 없이 새 인스턴스를 소환. 아래 검토 프롬프트만 전달.
   - **위 수단이 없는 경우**: 사용자에게 새 채팅 창을 열고 아래 프롬프트를 붙여넣도록 안내.

3. **검토 프롬프트** (이 내용만 새 인스턴스에 전달):

   ```
   [프로젝트 컨텍스트]
   {RPW 문서 전체 내용}

   [검토 대상]
   {diff 또는 코드 또는 설계안}

   [지시]
   위 변경사항을 검토하라.
   - 접근 방향 자체가 틀렸을 가능성을 열어두고 판단하라.
   - 논리적 비약, 누락된 엣지 케이스, 과도한 가정을 찾아라.
   - 칭찬 없이 문제점과 근거만 보고하라.
   - 발견 없으면 "발견 없음"으로만 끝내라.
   ```

4. **결과 통합**: 발견된 문제를 현재 세션으로 가져와 적용 여부를 판단하고, 채택·기각 결정을 RPW 문서에 기록한다.

## 규칙
- 이전 대화 내용을 검토 프롬프트에 포함하지 않는다. RPW + 검토 대상만.
- 검토자가 제안한 수정을 무비판적으로 적용하지 않는다 — 현재 맥락에서 타당성을 판단한 뒤 적용.
- 2회 순차 실행이 원칙 (첫 번째 검토 후 발견된 문제를 반영한 결과물을 두 번째 인스턴스가 재검토).
