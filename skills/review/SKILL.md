# Review — 셀프 코드 리뷰

`git diff` (스테이징 포함 시 `git diff HEAD`)를 기준으로 변경분만 리뷰한다.

## 점검 순서 (중요도순)

1. **정확성 버그**: 엣지 케이스(빈 입력, 경계값, None/0), off-by-one, 잘못된 조건, 자원 누수, 동시성.
2. **보안**: injection 류(SQL/command/path), 하드코딩된 시크릿, 검증 없는 외부 입력.
3. **기존 코드와의 정합성**: 같은 일을 하는 기존 함수를 중복 구현하지 않았는지, 호출부가 깨지지 않는지(시그니처 변경 시 전체 호출부 grep).
4. **단순화**: 불필요한 추상화, 죽은 코드, 요청 범위 밖 변경이 섞이지 않았는지.
5. **규칙 준수**: 프로젝트에서 읽기 전용으로 지정한 디렉토리(예: `BU/`, `thirdparty/`, 원본 데이터 폴더) 미수정, AI 흔적 없는 주석, 결과물 타임스탬프.

## 보고 형식

```
## 리뷰 결과
- [심각도: 높음/중간/낮음] 파일:라인 — 문제와 근거 한두 문장
```

- 발견 없음이면 "발견 없음"이라고만. 칭찬·요약 채우기 금지.
- 확신 없는 지적은 "확인 필요"로 표시하고 근거를 명시.
- 사용자가 수정까지 요청한 경우에만 고친다. 기본은 보고만.
