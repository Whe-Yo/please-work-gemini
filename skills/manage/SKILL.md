# Manage — 스킬 관리 및 업데이트

장착된 스킬을 주기적으로 점검하고 Harnessing_Agent 저장소의 최신 버전과 동기화한다.

---

## 일일 업데이트 확인 (boost에서 자동 트리거)

`~/.agents/harnessing_state.json`에서 마지막 확인 시각을 읽어 24시간 이상 경과 시 아래 절차를 실행한다.

1. **업데이트 확인**: Harnessing_Agent 저장소에서 원격 변경사항을 가져온다 (예: `git fetch`).
2. **변경 감지**: 로컬과 원격의 커밋 차이를 확인한다 (예: `git log HEAD..origin/main --oneline`).
3. **결과 분기**:
   - **변경 없음**: 확인 시각만 갱신하고 조용히 종료.
   - **변경 있음 (첫 세팅이 아닌 경우)**: 사용자에게 보고하고 선택을 요청한다.

     ```
     Harnessing_Agent에 업데이트가 있습니다.
     변경 내용: (커밋 요약 목록)

     업데이트하시겠습니까?
     - [업데이트]: 최신 버전으로 반영하고 현재 프로젝트 작업을 조정합니다.
     - [유지]: 다음 업데이트까지 현재 버전을 유지합니다.
     ```

4. **상태 저장**: `~/.agents/harnessing_state.json`에 확인 시각과 선택 결과를 기록한다.

```json
{
  "lastCheck": "YYMMDD_HHMM",
  "repoPath": "/path/to/Harnessing_Agent",
  "skipUntilNext": false
}
```

## 업데이트 실행 절차

사용자가 업데이트를 선택한 경우:

1. **풀**: `git pull`로 최신 버전을 받는다.
2. **스킬 diff**: 장착된 스킬과 저장소 최신 버전을 비교한다. 사용자가 직접 수정한 내용이 있으면 덮어쓰기 전에 diff를 보여준다.
3. **선택적 갱신**: 변경된 스킬만 갱신한다. 전체 덮어쓰기는 하지 않는다.
4. **RPW 작업 조정**: 현재 프로젝트 RPW 문서의 할 일 목록을 검토하고, 업데이트된 스킬·규칙과 충돌하거나 조정이 필요한 항목을 식별해 사용자에게 제안한다.
5. **완료 보고**: 갱신된 스킬 목록과 RPW 조정 내용을 요약한다.

## 수동 명령별 절차

### 현재 스킬 목록 확인
1. 에이전트의 스킬 폴더에 장착된 스킬을 나열한다.
2. `skills/SKILL_INDEX.md`와 대조해 미장착·구버전 스킬을 식별한다.

### 스킬 추가
1. `skills/SKILL_INDEX.md`에서 대상 스킬 설명을 확인한다.
2. 현재 환경에서 동작 가능한지 검토한다.
3. 스킬 폴더를 에이전트 스킬 디렉토리에 복사하거나 커맨드로 등록한다.

### 스킬 제거
1. 제거 대상 스킬을 삭제한다.
2. 제거 이유를 RPW 문서에 기록한다.

## 규칙
- boost와 worklog는 제거 요청 시 사용자에게 한 번 확인한다 (세션 연속성 핵심 스킬).
- `skipUntilNext: true` 상태에서는 다음 업데이트가 감지될 때까지 업데이트 확인 알림을 생략한다.
