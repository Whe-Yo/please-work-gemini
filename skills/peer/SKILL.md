# Peer — 외부 AI 인스턴스 위임

고차원 분석이 필요할 때 현재 에이전트와 분리된 별개의 AI 인스턴스에 분석을 위임한다.

## 위임 방법 (환경별)

- **서브에이전트 툴이 있는 경우** (예: Claude Code의 Agent 툴): 새 인스턴스를 직접 소환해 질문을 전달.
- **CLI 기반 에이전트**: 헤드리스 모드로 호출 (예: `claude -p "질문"`, `gemini "질문"`).
- **위 수단이 없는 경우**: 사용자에게 새 채팅 창을 열고 아래 프롬프트를 붙여넣도록 안내.

읽기 전용 분석이 목적이므로 외부 인스턴스에 파일 수정을 시키지 않는다 — "분석·제안만, 수정 금지"를 프롬프트에 명시.

## 질문 작성 규칙

좋은 답을 받으려면 질문에 다음을 포함한다:
1. **목표**: 무엇을 하려다 막혔는지
2. **현상**: 에러 메시지·로그 원문 (요약하지 말고 그대로)
3. **시도한 것**: 이미 실패한 접근과 실패 이유 (중복 제안 방지)
4. **제약**: 환경, 버전, 바꿀 수 없는 부분

## 결과 처리
- 외부 인스턴스의 답을 그대로 적용하지 말고, 현재 코드와 대조해 검증 후 적용한다.
- 채택한 결정은 RPW 문서에 WHAT/WHY/REJECTED로 기록하고 출처를 "Peer 분석"으로 남긴다.
- 응답 지연·오류가 발생하면 즉시 중단하고 사용자에게 보고한다. 재시도 루프 금지.

## 사용 기준
- 사용: 아키텍처 트레이드오프, 2회 이상 실패한 디버깅, 수학·알고리즘 검증
- 미사용: 단순 문법 질문, 검색으로 해결되는 것, 파일 읽으면 알 수 있는 것
