# Skill Index

에이전트가 첫 세팅(`setup` 스킬) 시 이 파일을 읽고 환경에 맞는 스킬을 자가 선택한다.
각 스킬의 전체 절차는 `skills/{name}/SKILL.md`에 있다.

---

## 핵심 스킬 (모든 환경 권장)

| 스킬 | 트리거 | 설명 |
|------|--------|------|
| `boost` | 새 세션 시작, "어디까지 했지" | 룰 파일·RPW 문서·git 히스토리를 읽고 이전 작업 맥락을 복원 |
| `worklog` | 작업 단위 완료, 결정·실패 직후 | RPW 문서(rule_plan_work.md 또는 work.md)를 표준 형식으로 실시간 갱신 |

## 품질 스킬

| 스킬 | 트리거 | 설명 |
|------|--------|------|
| `plan` | 파일 3개↑ 수정, 아키텍처 변경, "계획부터" | 구현 전 단계별 계획을 수립하고 사용자 승인을 받는다 |
| `verify` | 수정 완료 직후, "테스트해봐" | 변경사항이 실제로 동작하는지 실행으로 확인한다 |
| `review` | 커밋 전, "리뷰해줘" | git diff 기준으로 버그·보안·품질 문제를 점검한다 |
| `antithesis` | 코드·설계 완료 후, "반론 검토해줘" | 이전 대화 컨텍스트 없이 독립 인스턴스가 RPW만으로 검토 (오버피팅 방지) |

## 위임 스킬

| 스킬 | 트리거 | 설명 |
|------|--------|------|
| `peer` | 고난도 문제 2회 실패, "다른 AI한테 물어봐" | 별도 AI 인스턴스에 분석을 위임한다 |

## 특수 스킬

| 스킬 | 트리거 | 설명 |
|------|--------|------|
| `paper` | .tex/.bib 작업, "논문 봐줘" | 학술 논문 편집·LaTeX 컴파일 검증 |
| `resource` | 무거운 작업 착수 전, "뭐 돌고 있어", "GPU 비어?" | 호스트 리소스 점유를 실시간 조회해 보고 (저장 없음, 호스트별 분기) |

## 관리 스킬

| 스킬 | 트리거 | 설명 |
|------|--------|------|
| `setup` | 이 저장소 처음 적용 시 | 환경 파악 후 적합한 스킬을 자가 선택해 장착 |
| `manage` | "스킬 추가·업데이트·제거" | 장착된 스킬 목록을 주기적으로 점검하고 최신화 |
| `feedback` | 버그·개선점 발견, "피드백 남겨" | 사용 중 발견한 문제를 `log_for_test/`에 md로 기록·커밋(GitHub 이슈 아님) |
