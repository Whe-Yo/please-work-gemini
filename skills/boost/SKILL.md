# Boost — 세션 부트스트랩

새 세션에서 이전 작업 맥락을 복원한다. 아래 순서대로 수행:

1. 프로젝트 룰 파일(AGENTS.md, CLAUDE.md, GEMINI.md, .cursorrules 등 에이전트별 규칙 파일)이 있으면 읽는다.
2. RPW 문서(rule_plan_work.md 또는 work.md)를 읽는다 — 특히 최근 작업 이력, 할 일, `## AntiPatterns`, 의사결정 로그.
3. 버전 관리 히스토리와 현재 작업 트리 상태가 필요하다. 사용 가능한 버전 관리 도구로 최근 커밋 이력과 변경 파일 목록을 조회한다 (예: git — `git log --oneline -10`, `git status --short`). 버전 관리 도구가 없으면 이 단계는 건너뛴다.
4. 위 정보를 종합해 다음 형식으로 보고한다:

```
## 현황
- 마지막 작업: (work.md + 최근 커밋 기준 한두 문장)
- 작업 트리: (커밋 안 된 변경 유무와 핵심 파일)
- 다음 할 일: (work.md의 할 일 목록 상위 항목)
- 주의사항: (AntiPatterns나 미해결 이슈 중 현재 작업과 관련된 것만)
```

5. **하네싱 업데이트 확인**: `~/.agents/harnessing_state.json`을 읽어 마지막 확인 시각이 24시간 이상 경과했으면 `manage` 스킬의 "일일 업데이트 확인" 절차를 실행한다. 파일이 없거나 읽을 수 없으면, 또는 `manage` 스킬이 장착돼 있지 않으면 이 단계를 건너뛴다.

6. 보고 후 사용자 지시를 기다린다. 묻지 않고 작업을 시작하지 않는다.

## 규칙
- RPW 문서가 없으면 버전 관리 히스토리(있는 경우)만으로 요약하고, RPW 문서 생성을 제안한다.
- RPW 문서 내용이 코드 현재 상태와 다르면(예: 완료됐다고 적힌 기능이 실제로 없음) 그 불일치를 명시한다.
