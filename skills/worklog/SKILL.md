# Worklog — RPW 문서 관리

RPW 문서는 세션 간 기억을 잇는 유일한 수단이다. 프로젝트 루트에서 `rule_plan_work.md` 또는 `work.md`를 찾아 사용한다. 둘 다 없으면 `rule_plan_work.md`로 새로 생성한다. 다음 구조를 유지한다:

```markdown
# work.md

## 현재 상태
(지금 어디까지 됐는지 2-3문장. 갱신 시 덮어씀)

## 할 일
- [ ] (우선순위순)

## 작업 로그
### 2026-06-10
- (한 일, 결과, 관련 커밋)

## 의사결정 로그
### [2026-06-10] (결정 제목)
- WHAT: 결정 내용
- WHY: 선택 이유
- REJECTED: 기각된 대안과 이유

## AntiPatterns
- [2026-06-10] (증상) → (원인) → (해결/회피법). 재현 조건 명시.
```

## 갱신 규칙
- **언제**: 주요 작업 단위 완료 시, 비자명한 결정 직후, 실패한 접근을 폐기할 때. 사소한 수정마다 갱신하지 않는다.
- **날짜는 절대 날짜**: "오늘", "어제" 금지. `YYYY-MM-DD`.
- **덮어쓰기 vs 추가**: "현재 상태"와 "할 일"은 덮어쓰고, 로그 섹션들은 위에 추가(최신이 위).
- **검증**: 갱신 전 기존 내용을 읽고, 완료된 할 일은 체크 처리. 코드 현재 상태와 모순되는 기술은 수정한다.
- 파일이 없으면 위 템플릿으로 생성한다.
