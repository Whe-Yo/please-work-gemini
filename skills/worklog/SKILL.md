# Worklog — RPW 문서 관리

RPW는 히스토리 로그가 아닌 **현재 상태 스냅샷**이다. 프로젝트 루트에서 `rule_plan_work.md`를 찾아 사용한다. 없으면 새로 생성한다.

```markdown
# RPW — [프로젝트명]

## Rule
### Always do
- (글로벌 규칙 + 프로젝트별)

### Ask first
- (프로젝트별)

### Never do
- (프로젝트별)

---

## Plan
목표: (현재 작업 단위의 목표 한 줄)
- [ ] 세부 작업

---

## Work
(지금 어디까지 됐는지 2-3문장)
```

## 갱신 규칙
- **언제**: 작업 단위 전환 시, Plan 목표 달성 시, Work 상태가 바뀔 때. 사소한 수정마다 갱신하지 않는다.
- **덮어쓰기**: Rule·Plan·Work 모두 덮어씀. 이전 내용 보존 불필요.
- **검증**: 갱신 전 기존 내용을 읽고 현재 상태와 모순되는 내용을 수정한다.
- 파일이 없으면 위 구조로 생성한다.
