# Resource — 착수 전 리소스 스냅샷

여러 프로젝트를 한 호스트에서 돌릴 때, 무거운 작업(학습·실험·장시간 서버) 착수 전에 **지금 무엇이 호스트 자원을 점유하는지** 실시간으로 확인한다.

핵심 원칙: **측정만 한다, 저장하지 않는다.** "실행 중" 목록을 파일에 적어두면 즉시 현실과 어긋난다(stale·좀비). 항상 라이브 조회한다.

## 절차

1. **환경 분기로 라이브 조회** (가용한 것만):
   - NVIDIA GPU: `nvidia-smi` — VRAM 사용량·점유 프로세스
   - 무거운 CPU/메모리 프로세스: Linux `ps -Ao pid,pcpu,pmem,comm --sort=-pcpu | head`, macOS `ps -Ao pid,%cpu,%mem,comm -r | head`
   - Apple Silicon GPU 부하: `powermetrics`(권한 필요 시 생략하고 안내)
   - 점유 포트(서버 띄울 때): `lsof -i -P -sTCP:LISTEN` 또는 `ss -tlnp`
2. **요약 보고**: 무엇이 얼마나 점유 중인지 + 착수하려는 작업의 예상 요구치와 비교해 "지금 착수 / 끝날 때까지 대기" 판단 근거를 제시한다.
3. **사용자 결정**: 자동으로 띄우거나 죽이지 않는다. 보고 후 사용자가 결정한다.

## 규칙
- 읽기 전용 조회만. 프로세스를 죽이거나 우선순위를 바꾸지 않는다.
- "실행 중" 상태를 파일에 저장하지 않는다 — 매 호출 시 실시간 조회.
- 조회 도구가 환경에 없으면(예: Mac에 `nvidia-smi` 없음) 건너뛰고 가용한 도구로 대체한다.

## 범위 밖 (의도적 제외)
대기열·우선순위 큐·자동 디스패치는 넣지 않는다. 라이브 조회만으로 자원 경합이 반복 확인된 뒤에 별도 기능으로 입증한다. (큐는 다중 에이전트 동시 쓰기 레이스·좀비 항목·전이 트리거 부재 문제를 동반하므로, 검증 전 도입하지 않는다.)
