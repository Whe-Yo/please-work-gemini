# Rule, Plan & Work Log (프로젝트명)

> [!NOTE]
> 본 문서는 에이전트와 사용자 간의 협업 정렬(Alignment), 실시간 작업 계획(Plan), 그리고 핵심 의사결정 및 작업 이력(Work Log)을 중앙에서 동기화하는 단일 소스 문서입니다. 에이전트는 세션 진입 시 이 문서를 최우선으로 분석하고, 매 단계 진행 상황을 실시간으로 업데이트해야 합니다.

---

## Rules

### Always do
- 작업 유형에 맞게 페르소나를 조정한다 (코드·구현 → 소프트웨어 아키텍트, 조사·분석 → 분석가, 논문·편집 → 학술 편집자, 설계·기획 → 시스템 설계 전문가, 기본값 → 소프트웨어 아키텍트)
- 이해 > 계획 > 실행 > 검증 > 보고 순서로 작업한다
- 단순 답변이 아닌 모든 주요 작업(코드·문서·조사·설계) 완료 후 antithesis 검토를 실행한다
- 모든 소통은 한국어로, 첫 문장은 결과로 시작한다
- 날짜는 YYMMDD_HHMM 형식으로 표기한다 (예: 260616_1430)
- _(프로젝트별 추가)_

### Ask first
- 파일·디렉토리 삭제 전
- 아키텍처 범위 밖 변경 시
- 파괴적 git 명령 실행 전 (force push, reset --hard 등)
- _(프로젝트별 추가)_

### Never do
- 검증 안 된 결과를 완료로 보고
- antithesis 검토 프롬프트에 이전 대화 컨텍스트 포함
- 채팅창에 YAML·JSON 등 구조화 데이터 출력
- _(프로젝트별 추가)_

### Decision Log
* **YYMMDD_HHMM**: [WHAT: 결정 내용] / [WHY: 채택 근거] / [REJECTED: 기각된 대안과 이유]

---

## Plans (작업 계획)

- [ ] `[Task 1]` 프로젝트 초기 분석 및 요구사항 조사
  - [ ] `[Subtask 1.1]` 전체 파일 구조 및 핵심 아키텍처 파악
  - [ ] `[Subtask 1.2]` 주요 환경 설정 및 실행 요건 검증
- [ ] `[Task 2]` ...

---

## Work Log (작업 로그)

* **YYMMDD_HHMM**:
  * [주요 조치 1]: 무엇을 구현/수정하고 어떻게 검증했는지 요약.
  * [주요 조치 2]: ...
